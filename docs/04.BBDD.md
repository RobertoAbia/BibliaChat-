## **B) Constraints y reglas de integridad (Documentación)**

**Foreign Keys (FK)**

* `user_profiles.user_id` → `auth.users.id` (implícito: 1–1 con Supabase Auth)

* `user_profiles.bible_version_code` → `bible_versions.code`

* `user_devices.user_id` → `auth.users.id`

* `user_entitlements.user_id` → `auth.users.id`

* `chats.user_id` → `auth.users.id`

* `chats.topic_key` → `chat_topics.key`

* `chat_messages.chat_id` → `chats.id`

* `saved_messages.user_id` → `auth.users.id`

* `saved_messages.chat_message_id` → `chat_messages.id`

* `daily_verse_texts.verse_date` → `daily_verses.verse_date`

* `daily_verse_texts.bible_version_code` → `bible_versions.code`

* `devotion_variants.devotion_date` → `devotions.devotion_date`

* `user_devotions.user_id` → `auth.users.id`

* `user_devotions.devotion_date` → `devotions.devotion_date`

* `daily_activity.user_id` → `auth.users.id`

* `plan_days.plan_id` → `plans.id`

* `user_plans.user_id` → `auth.users.id`

* `user_plans.plan_id` → `plans.id`

* `user_plan_days.user_plan_id` → `user_plans.id`

* `user_points.user_id` → `auth.users.id`

* `user_badges.user_id` → `auth.users.id`

* `user_badges.badge_key` → `badges.key`

**Unique / PK importantes**

* `daily_verse_texts` tiene PK compuesta: `(verse_date, bible_version_code)`

* `devotion_variants` tiene PK compuesta: `(devotion_date, variant_key)`

* `daily_activity` tiene PK compuesta: `(user_id, activity_date)`

* **Business rule:** `UNIQUE(user_devotions.user_id, user_devotions.devotion_date)` (un registro por usuario y día)

## **C) Notas funcionales clave (timezone / día local)**

* `user_profiles.timezone` es **IANA timezone** (ej. `"America/New_York"`), detectada desde el dispositivo en onboarding.

* Todas las métricas diarias (**racha, daily\_activity, badges por racha**) se calculan con **fecha local del usuario** (timezone), no en UTC.

* El backend (Edge Function o lógica server-side) debe convertir timestamps a la fecha local del usuario antes de evaluar lógica diaria.
