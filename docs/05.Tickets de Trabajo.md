# **tickets.md — Biblia Chat (MVP)**

**Arquitectura:** Supabase-first (Postgres \+ Auth \+ RLS) \+ Edge Functions para IA/Billing/operaciones sensibles  
 **App:** Flutter iOS/Android  
 **Monetización:** Paid-first con trial 3 días (mensual)  
 **Chat:** múltiples hilos por usuario, 1 hilo por topic en MVP  
 **IA:** OpenAI stateless \+ `user_profiles.ai_memory` \+ `chats.context_summary`

---

## **Convenciones**

* **Prioridad**

  * **P0**: bloqueante / core MVP

  * **P1**: muy importante pero no bloquea el core

  * **P2**: polish / nice-to-have MVP

* **DoD (Definition of Done)**: criterio verificable para cerrar ticket

* **Depende de**: tickets que deben estar completados antes

### **Arquitectura Técnica de Monetización (referencia)**

**Edge Functions requeridas:**

| Función | Ticket | Descripción |
|---------|--------|-------------|
| `handle-purchase` | T-0403 | Guardar `rc_app_user_id` en user_profiles al comprar |
| `restore-purchases` | T-0403b | Migrar datos del usuario antiguo al nuevo |
| `revenuecat-webhook` | T-0404 | Recibir eventos de compra/cancelación/renovación |

**Flujo de COMPRA (usuario anónimo):**
```
Flutter → RevenueCat SDK → Compra exitosa en App Store/Google Play
       → Llama Edge Function "handle-purchase"
       → Guarda rc_app_user_id en user_profiles
       → Usuario es premium (sin registro)
```

**Flujo de RESTAURACIÓN (tras reinstalar):**
```
Flutter → RevenueCat SDK → "Restore purchases"
       → Obtiene rc_app_user_id del receipt (vinculado a Apple ID/Google)
       → Llama Edge Function "restore-purchases"
       → Edge Function:
           1. Busca perfil anterior con ese rc_app_user_id
           2. Si existe: migra chats, progreso, badges, etc. al usuario actual
           3. Actualiza rc_app_user_id en perfil actual
       → Usuario recupera premium + TODOS sus datos
       → Experiencia transparente: solo pulsó "Restaurar compras"
```

**Datos a migrar en restore-purchases:**
- `user_profiles` (perfil, ai_memory, preferencias)
- `chats` + `chat_messages` (historial completo)
- `saved_messages` (mensajes guardados)
- `user_plans` + `user_plan_days` (progreso en planes)
- `daily_activity` (rachas)
- `user_badges` + `user_points` (gamificación)
