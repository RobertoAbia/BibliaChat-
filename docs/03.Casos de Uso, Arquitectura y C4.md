# Casos de Uso

Aquí van **5 casos de uso** (Use Cases) bien definidos, en lenguaje de **arquitectura \+ DDD/Clean**, alineados con tu PRD y tu modelo **paid-first con trial**.

---

## **1\) Iniciar app con sesión anónima y crear perfil mínimo**

**Actor primario:** Usuario  
 **Objetivo:** Entrar sin fricción y crear identidad persistente en backend.

**Precondiciones**

* App instalada por primera vez.

* No existe sesión activa.

**Flujo principal**

1. App llama a `supabase.auth.signInAnonymously()`.

2. Backend/DB crea `auth.users`.

3. Se crea/asegura `user_profiles` ligado a `user_id`.

4. Se devuelve sesión y `user_id` estable para el resto del sistema.

**Postcondiciones**

* Existe `user_id` único y persistente.

* Existe `user_profiles` inicial (aunque incompleto).

**Agregados/Bounded Context**

* **Identity & Access**: `User`

* **Profile**: `UserProfile`

---

## **2\) Completar onboarding y persistir preferencias**

**Actor primario:** Usuario
 **Objetivo:** Personalización inicial de toda la experiencia.

**Precondiciones**

* Usuario autenticado (anónimo o registrado).

* Existe `user_profiles`.

**Flujo principal**

1. Usuario responde secuencia de 10 preguntas:
   - Edad (age_group)
   - Género (gender)
   - País/Origen (origin)
   - Denominación (denomination)
   - Versión Biblia (bible_version_code)
   - Motivo (motive)
   - Recordatorio (reminder_enabled, reminder_time)
   - Persistencia (persistence_self_report)
   - Mensaje del corazón (first_message)

2. App valida y guarda progresivamente en `user_profiles`.

3. Al completar, se auto-detecta timezone del dispositivo.

4. Se marca `onboarding_completed = true` en `user_profiles`.

5. Usuario llega a HOY.

**Postcondiciones**

* `user_profiles` completo con preferencias.

* App lista para generar contenido personalizado.

**Agregados/Bounded Context**

* **Profile**: `UserProfile`

* **Engagement**: `OnboardingSession` (conceptual/eventos)

---

## **3\) Iniciar trial paid-first y habilitar acceso premium**

**Actor primario:** Usuario  
 **Objetivo:** Activar trial de 3 días (mensual) y desbloquear la app.

**Precondiciones**

* Onboarding completado (o punto de paywall definido).

* Usuario con `user_id` válido.

* RevenueCat configurado con `appUserID = supabaseUserId`.

**Flujo principal**

1. Usuario ve paywall (trial 3 días \+ precio).

2. Usuario inicia compra (IAP).

3. RevenueCat confirma entitlement (trial activo).

4. App sincroniza estado premium (`is_premium = true`) para UX y gating.

5. Se registran eventos `trial_started`, `premium_status_updated`.

**Postcondiciones**

* Trial activo y features premium habilitadas.

* Estado premium consultable en app y backend.

**Agregados/Bounded Context**

* **Billing**: `Subscription`, `Entitlement`

* **Identity**: `User`

---

## **4\) Crear/Reutilizar chat por tema y enviar mensaje a IA (hilo \+ persistencia)**

**Actor primario:** Usuario
 **Objetivo:** Conversación por tema con continuidad, persistida en Supabase.

**Precondiciones**

* Usuario autenticado.

* Usuario free: debe tener mensajes disponibles (≤5/día). Usuario premium: sin límite.

* `user_profiles` disponible.

**Flujo principal**

1. Usuario selecciona un tema (ej. `ansiedad_miedo`) o "Otro", o inicia chat libre.

2. Sistema busca chat existente del usuario con ese `topic`; si no existe, crea uno.

3. **[NUEVO]** Sistema verifica límite de mensajes:
   - Si usuario es premium → continúa sin restricción
   - Si usuario free → verifica `daily_activity.messages_sent < 5`
   - Si límite alcanzado → muestra diálogo con opción "Ver planes"

4. Usuario envía mensaje.

5. Backend construye contexto:

   * Prompt base \+ prompt dinámico \+ `user_profiles.ai_memory` (si existe) \+ `context_summary` \+ últimos N mensajes.

6. Backend llama a OpenAI y recibe respuesta.

7. Sistema guarda:

   * mensaje del usuario en `chat_messages`

   * respuesta IA en `chat_messages`

   * actualiza `chats.last_message_at` y `last_message_preview`

   * **[NUEVO]** incrementa `daily_activity.messages_sent` (solo usuarios free)

   * opcional: actualiza `context_summary` y `ai_memory` (si aplica).

8. Devuelve respuesta al cliente.

**Postcondiciones**

* Conversación persistida y recuperable por hilo/tema.

* Historial visible en "Mis chats".

* **[NUEVO]** Contador de mensajes diarios actualizado (usuarios free).

**Agregados/Bounded Context**

* **Conversation**: `ChatThread (chats)`, `ChatMessage (chat_messages)`

* **Profile**: `UserProfile (ai_memory)`

* **AI Orchestration** (servicio de aplicación): `ChatCompletionService`

---

## **5\) Completar actividad diaria (HOY) y actualizar racha/calendario**

**Actor primario:** Usuario  
 **Objetivo:** Refuerzo de hábito y retención.

**Precondiciones**

* Usuario autenticado.

* Acceso a HOY y contenido diario.

**Flujo principal**

1. Usuario abre HOY.

2. El sistema muestra racha y calendario semanal.

3. Usuario completa al menos una acción válida:

   * lee devoción/versículo (según reglas de tracking) **o**

   * envía mensaje en chat.

4. Se registra “día completado” y se recalcula racha (incremento/reset).

5. Se actualiza estado para UI y analytics.

**Postcondiciones**

* Día marcado como completado.

* Racha actualizada.

* Evento de analytics registrado.

**Agregados/Bounded Context**

* **Engagement**: `Streak`, `DailyCompletion`

* **Content**: `DailyVerse`, `Devotion`

---

* Escalabilidad progresiva sin migraciones

---

## **2\. DECISIÓN ESTRATÉGICA PRINCIPAL**

### **🔹 Supabase-First Architecture**

**Supabase es la fuente de verdad del sistema.**

No existe backend dedicado en el MVP.

* Base de datos: PostgreSQL (Supabase)

* Auth: Supabase Auth (anónimo \+ upgrade)

* Seguridad: RLS

* Backend lógico: Supabase Edge Functions (Deno/TypeScript)

* IA: OpenAI GPT-5.2 (stateless)
  * Usa `role: "developer"` en lugar de `role: "system"`
  * Usa `max_completion_tokens` en lugar de `max_tokens`

* Pagos: RevenueCat

👉 **El backend "vive" dentro de Supabase.**

---

## **5\. MODELO DE CONVERSACIÓN**

### **Reglas clave**

* Un usuario puede tener **varios chats** (híbrido: por tema o libres)

* OpenAI es **stateless**

* Supabase guarda todo el historial

### **Modelo Freemium**

* **Usuarios free:** 5 mensajes/día (reset a medianoche)
* **Usuarios premium:** Chat ilimitado
* Contador en `daily_activity.messages_sent`
* Al agotar mensajes → Paywall con opciones de suscripción

### **Contexto enviado a IA (orden fijo)**

1. Prompt base del sistema

2. Prompt dinámico (denominación, origen…)

3. `user_profiles.ai_memory`

4. `chats.context_summary`

5. Últimos 12 mensajes

## **11\. SEGURIDAD Y PRIVACIDAD**

### **Reglas RLS obligatorias**

1. `chats`: solo owner

2. `chat_messages`: solo owner \+ chat propio

3. `user_profiles`: solo owner (incluye `ai_memory`)

### **Otras reglas**

* IA no inventa datos

* OpenAI keys solo server-side

* Borrado completo de datos al eliminar cuenta
